Ce qu'il reste à construire

Composant Outil Description
2 questions d'aiguillage React (site) 2 écrans simples avec 2 boutons chacun, avant la redirection Forms
Écran identification (Chemin C) React (site) Mini-formulaire AVS + email avec bouton "Vérifier"
Flux "Vérification identité" Power Automate HTTP trigger → query SP par AVS → retourne JSON {existe, prenom, historique}
Flux "Envoi lien raccourci" Power Automate Génère URL Forms pré-rempli → envoie par email au stagiaire
Forms raccourci Microsoft Forms Version allégée : seulement secteur, dates, signature, remarques (~10 q.)
Anti-doublon référent Power Automate (Flux 1) Avant créer Référent : query par email. Si existe → réutiliser l'ID

Point de sécurité important

Le formulaire raccourci (Chemin C) n'est jamais visible sur le site. Son URL est envoyée uniquement par email après vérification AVS + email. Même si le Forms reste techniquement "public" (pas de Azure AD), le lien n'est pas devinable et contient des paramètres pré-remplis uniques. C'est un compromis réaliste : les stagiaires externes n'ont pas de compte Microsoft, mais le lien par email ajoute une couche de sécurité.
