Architecture Frontend — Clair-Bois Calendrier

Guide visuel a destination des developpeurs juniors. Stack : React 19 + Vite + Tailwind CSS v4. Heberge sur GitHub Pages.

index.html — Le squelette
HTML

Fichier HTML minimal situe a la racine du projet frontend/. Il contient :

  • Un <div id="root"> vide — c'est ici que React va injecter toute l'application
  • Un <script type="module" src="/src/main.jsx"> qui charge le point d'entree JavaScript
  • Les polices Google Fonts (Open Sans) en preconnect
  • Le lang="fr" pour l'accessibilite

En production, Vite remplace ce fichier par une version optimisee avec les noms de fichiers hashes.

▶
main.jsx — Le demarrage React
JSX

Ce fichier fait exactement 3 choses :

  • Importe createRoot depuis React 19 (nouvelle API de rendu)
  • Importe le fichier index.css (Tailwind + theme personnalise)
  • Monte <App /> dans #root avec StrictMode (verifications supplementaires en dev)
import { StrictMode } from 'react' import { createRoot } from 'react-dom/client' import App from './App.jsx' createRoot(document.getElementById('root')).render( <StrictMode><App /></StrictMode> )
▶
App.jsx — Le composant racine
JSX

C'est le chef d'orchestre. Au montage, il :

  • Lance un fetch() vers planning.json (via useEffect, une seule fois)
  • Transforme les donnees brutes avec transformPlanningData()
  • Affiche un spinner pendant le chargement, un message si erreur
  • Gere toute la navigation via useState('home') — pas de React Router
  • Rend le composant correspondant a l'ecran actif (rendu conditionnel)

Pourquoi pas de React Router ?

L'application est un site statique heberge sur GitHub Pages. La navigation est geree entierement par l'etat React (currentView), ce qui evite la configuration de redirections serveur. C'est un choix de simplicite adapte a la taille du projet.

2 React dans tout ca ?

Le role de React, le Virtual DOM, et l'arbre des composants de l'application.

▶
React en 30 secondes
Concept

React est une bibliotheque JavaScript pour construire des interfaces. Trois concepts cles :

  • Composants — Chaque ecran, chaque bout d'interface est un composant reutilisable (une fonction qui retourne du JSX).
  • Virtual DOM — React maintient une copie virtuelle de la page. Quand l'etat change, il calcule le minimum de modifications a appliquer au vrai DOM. C'est ce qui rend les mises a jour rapides.
  • Single Page App (SPA) — Le navigateur charge index.html une seule fois. Ensuite, la navigation se fait sans rechargement : React remplace les composants a l'ecran.

Format plat (Power Automate) vs hierarchique (React)
Donnees

En entree — Le JSON genere par Power Automate est un tableau plat de creneaux :

{ "creneaux": [ { "etablissement": "Blanchisserie Tourbillon", "secteur": "Lavage", "dateDebut": "2026-03-01", "dateFin": "2026-03-05", "placesTotal": 3, "placesUtilisees": 0 }, ... ], "modulesMetiers": { ... } }

En sortie — transformPlanningData() produit une structure hierarchique :

{ "etablissements": [ { "name": "Blanchisserie Tourbillon", "secteurs": [ { "name": "Lavage", "weeks": [ { "weekNumber": 10, "status": "available", "totalSlots": 3, "usedSlots": 0 } ] } ] } ], "modulesMetiers": { ... } }

Cela permet aux composants React de parcourir facilement : data.etablissements.map(e => e.secteurs.map(...))

Rendu conditionnel — Pas de React Router

La navigation repose sur un simple switch implicite via des blocs {currentView === 'xxx' && <Composant />}. Quand currentView change, React demonte l'ancien composant et monte le nouveau. Chaque composant recoit les donnees necessaires via ses props et les callbacks de navigation (onBack, onGoHome, etc.).

Chemins d'aiguillage et sections
Config

Le fichier formConfig.js definit de maniere declarative quelles etapes sont visibles pour chaque chemin. La cle est construite ainsi :

// Exemples de cles : 'stages-moi-non' // Stage, pour moi, jamais inscrit 'stages-autre-oui' // Stage, par un referent, deja inscrit 'modules-moi' // Modules, pour moi-meme 'modules-autre' // Modules, par un referent
CheminSections affichees
stages-moi-nonStagiaire → Curatelle → Urgence → AI → Complements → Declaration
stages-autre-nonReferent → Stagiaire → Curatelle → Urgence → AI → Complements → Declaration
stages-moi-ouiStagiaire (retour simplifie)
stages-autre-ouiReferent → Stagiaire → Curatelle → Urgence → AI → Complements → Declaration
modules-moiStagiaire → Curatelle → Urgence → AI → Complements → Declaration
modules-autreReferent → Stagiaire → Curatelle → Urgence → AI → Complements → Declaration

Validation des champs (validation.js)
Validation

Chaque fonction retourne { valid: boolean, message: string }. Les validations metier specifiques a la Suisse :

ValidateurRegle
validateAVS()Format 756.XXXX.XXXX.XX — 13 chiffres, commence par 756
validatePhone()+41XXXXXXXXX (12 car.) ou 0XXXXXXXXX (10 car.)
validateNPA()4 chiffres entre 1000 et 9999 (code postal suisse)
validateEmail()Format email standard
validateDateNaissance()Date valide, personne de 14 a 99 ans
validateRequired()Champ non vide

La validation est effectuee champ par champ avec retour visuel immediat, puis globalement avant le passage a l'etape suivante.

Le payload JSON contient un champ cheminKey (ex: "stages-moi-non") qui permet au flux Power Automate de router la logique backend selon le type d'inscription.

Pourquoi un formulaire integre plutot que Microsoft Forms ?

Microsoft Forms ne permet pas : la validation du numero AVS, la logique conditionnelle avancee, le pre-remplissage de modules+semaines, ni la maintenance simple (8 formulaires distincts a synchroniser). Le formulaire React integre offre tout cela avec un seul composant et un seul flux backend.

Vite — Le bundler
Build

Vite est l'outil de build et de developpement. Voici sa configuration (vite.config.js) :

import { defineConfig } from 'vite' import react from '@vitejs/plugin-react' import tailwindcss from '@tailwindcss/vite' export default defineConfig({ plugins: [react(), tailwindcss()], base: '/clair-bois-calendrier/', })

Trois roles principaux :

  • Developpement — Serveur local avec HMR (Hot Module Replacement) : les modifications s'affichent instantanement sans rechargement
  • Build — npm run build produit un dossier /dist avec les fichiers optimises (minifies, hashes, tree-shaken)
  • Base path — base: '/clair-bois-calendrier/' est critique : il indique que le site n'est pas a la racine mais dans un sous-dossier de GitHub Pages. Sans cela, tous les imports seraient casses.
▶
GitHub Actions — Le deploiement automatique
CI/CD

A chaque git push sur la branche principale, GitHub Actions execute automatiquement :

  • Installation des dependances (npm ci)
  • Build du projet (npm run build)
  • Deploiement du contenu de /dist sur GitHub Pages

Le site est accessible a l'URL : https://rochat-dev.github.io/clair-bois-calendrier/
